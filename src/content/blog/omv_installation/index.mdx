## OpenMediaVault ?

OpenMediaVault (OMV) est un système d'exploitation basé sur Debian, conçu pour créer et gérer un serveur de stockage en réseau (NAS). Il vous permet de partager et sauvegarder vos fichiers, avec des protocoles comme SMB/CIFS, FTP, NFS et SSH. Cette solution offre une interface web intuitive pour la gestion des disques, des utilisateurs, des permissions et des services réseau. Il supporte les plugins pour ajouter des fonctionnalités et aussi la conteneurisation avec Docker. OMV est facile à installer et à configurer sans nécessiter de grandes compétences techniques.

### Installation d'OMV

*Dans cet article, je vais installer la solution OpenMediaVault en tant que machine virtuelle Proxmox, mais le principe reste le même pour une installation en barre métal.*

1. Récupérer la dernière version de [OpenMediaVault](https://www.openmediavault.org/download.html),
2. Importer l'ISO dans les ISOs Proxmox ou ajouter l'ISO sur [votre clé Ventoy](/blog/ventoy_installation_update/) (pour une installation sans Proxmox)
3. Pour une installation Barre Métal, suivez le tuto sur l'[installation d'un OS sur le HP Gen 8](/blog/hp_gen_8_mise_en_place/) ou suivez celui de l'[ajout d'une VM dans Proxmox](/blog/proxmox_installation/).

L'installation de OMV se déroule comme n'importe quelle [installation de Debian](/blog/linux_debian_12_installation/) dont vous avez un exemple [ici](/blog/linux_debian_12_installation/).

### Ajouter vos disques dur en "Passthrought" (Uniquement sur Proxmox)

Si vous avez installé OMV depuis une VM Proxmox, vous ne voyez pas les disques dur et c'est normale. Nous allons attribuer les disques à OMV sans passer par Proxmox. Cela s'appelle faire un Passthrough (passer au travers).

Pour les installations Barre Metal, vous devez voir les disques directement depuis OMV.

Nous allons commencer par lister les disques présents sur le système.

1. Rendez-vous dans l'interface de votre nœud Proxmox puis dans le Shell.
2. Lancer la commande `fdisk -l` pour lister tous les disques présents et récupérer les chemins `/dev/sdx` des disques à attribuer à votre OMV.

<Alert type="danger">Il y a votre disque système alors ne vous trompez pas</Alert>

3. Nous allons récupérer l'UUID des disques avec la commande suivant `lsblk -l -o NAME,PARTUUID /dev/sdaX` qui doit vous retourner quelques chose comme ça.

```shell
root@pve:~# lsblk -l -o NAME,PARTUUID /dev/sda
NAME PARTUUID
sda  
sda1 a8d0f94d-c351-7c45-924d-84f19df1fa64
root@pve:~# lsblk -l -o NAME,PARTUUID /dev/sdb
NAME PARTUUID
sdb  
sdb1 b33d9831-01
root@pve:~# lsblk -l -o NAME,PARTUUID /dev/sdc
NAME PARTUUID
sdc  
sdc1 4108a20d-9f6c-2f41-92f4-3c956e37c6d3
root@pve:~# lsblk -l -o NAME,PARTUUID /dev/sdd
NAME PARTUUID
sdd  
sdd1 f554a53d-01
```

Une fois les UUID récupérés, nous allons déclarer les disques dans notre VMs avec la commande `qm set <VMID> --scsi0 /dev/disk/by-id/<ton-disque>`.

* `<VMID>` : Correspond à l'ID de la VM concernée.
* `<ton-disque>` :  Correspond à `sdaX`, `sdbX`, etc.
* `--scsiX` : doit être incrémenté en fonction du nombre de disques (pas deux fois le même nombre).

<Alert type="info">

\* `--scsiX` : Plus flexible et recommandé, il permet le hot-plug, par exemple, mais limité à 15 par VM.

\*  `--sataX` : Simple à utiliser, mais limité à 5 par VMs,

\*  `--ideX` : Utilisé pour la compatibilité avec les systèmes plus anciens.

\*  `--virtioX` : Recommandé pour des performances optimales dans des environnements virtualisés.

\*  `--usbX` : Pour attacher des périphériques USB physiques.

</Alert>

```
qm set 130 --scsi1 /dev/disk/by-id/a8d0f94d-c351-7c45-924d-84f19df1fa64
qm set <VMID> --scsi2 /dev/disk/by-id/b33d9831-01
qm set <VMID> --scsi3 /dev/disk/by-id/4108a20d-9f6c-2f41-92f4-3c956e37c6d3
qm set <VMID> --scsi4 /dev/disk/by-id/f554a53d-01
```

Vérifiez en vous rendant dans 

Appuyer sur <kbd>F11</kbd>,

* Sélectionner le numéro de votre clé (Attention le pavé numérique ne fonctionne pas, il faut utiliser la <kbd>MAJ</kbd>+<kbd>1,2,3,4,...</kbd>,
* Sélectionnez votre distribution (ici OpenMediaVault),
* Suivez les étapes **ATTENTION** de bien sélectionner le bon disque dur et de mettre un mot de passe `root` solide (*il sert pour la connexion SSH en root*).

Une fois l'installation terminée, débrancher la clé USB et redémarrer.

Ensuite, rendez-vous sur l'adresse IP de votre serveur (pas celle de l'iLO) et connectez-vous avec le couple login/mot de passe par défaut **admin/openmediavault**.
